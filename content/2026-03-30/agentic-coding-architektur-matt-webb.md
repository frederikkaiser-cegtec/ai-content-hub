## Kernaussagen
- Matt Webb (Tech-Architekt): KI-Agents loesen jedes Problem durch Brute Force — sie "schleifen Probleme zu Staub"
- Das reicht aber nicht: Code muss wartbar, adaptiv und komponierbar sein
- Erkenntnis: "Ich schaue weniger auf Code als je zuvor und denke mehr ueber Architektur nach als je zuvor"
- Der echte Wert liegt in Libraries mit guten Interfaces, die den richtigen Weg zum einfachen Weg machen
- Vibe Coding (nicht Coding, nicht Vibe Coding) verschiebt den Skill von Implementierung zu Systemdesign

## Relevanz fuer Marketing
Die Parallele zu Marketing-Operations ist direkt: Auch dort loesen KI-Tools einzelne Aufgaben — aber ohne Architektur (CRM-Struktur, Datenfluss, Prozessdesign) entsteht nur teurer Wildwuchs. B2B-Teams brauchen jemanden, der das Gesamtsystem denkt.

## Post-Idee
Hook: "KI-Agents koennen jedes Problem loesen. Das ist das Problem."
Angle: Matt Webbs Insight ueber Agentic Coding gilt 1:1 fuer Marketing-Ops. Wenn jeder Agent jede Aufgabe loesen kann, wird die Frage nicht "Welches Tool?", sondern "Welche Architektur?". Drei Fragen, die sich jedes B2B-Team stellen sollte, bevor es den naechsten Agent einrichtet. Schluss-These: Der neue Marketing-Skill ist nicht Prompting — sondern System-Design.
